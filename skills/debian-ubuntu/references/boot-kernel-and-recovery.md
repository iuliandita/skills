# Boot, Kernel, and Recovery

Use this for GRUB, initramfs, EFI, kernel packages, and boot recovery on Debian/Ubuntu systems.
On Debian and Ubuntu, GRUB config is regenerated by `update-grub` (a wrapper around
`grub-mkconfig`), and the initramfs is built by `initramfs-tools` via `update-initramfs`. These two
must agree with the installed kernel metapackage.

## Inspect first

```bash
findmnt /boot
findmnt /boot/efi
uname -r
dpkg -l 'linux-image-*' | grep '^ii'
dpkg-query -W -f='${Package}\n' 'linux-image-generic*' 'linux-image-amd64' 2>&1 || true
ls -1 /boot/vmlinuz-*
[ -d /sys/firmware/efi ] && echo "EFI boot" || echo "BIOS/legacy boot"
efibootmgr -v 2>&1 || true
cat /etc/default/grub
```

- `findmnt /boot/efi` confirms the ESP is mounted (usually `/boot/efi`, FAT32). If it is not
  mounted, do not reinstall GRUB - mount it first.
- `[ -d /sys/firmware/efi ]` is the reliable EFI-vs-BIOS test. EFI and BIOS GRUB packages differ
  (`grub-efi-amd64` vs `grub-pc`); installing the wrong one breaks boot.
- The kernel metapackage (`linux-image-generic`, `linux-image-generic-hwe-*` on Ubuntu, or
  `linux-image-amd64` on Debian) pulls in the actual kernel. Know which one the system tracks before
  removing kernels.

## Regenerate config (the two subsystems)

```bash
sudo update-grub                      # regenerates /boot/grub/grub.cfg from /etc/default/grub + /etc/grub.d
sudo update-initramfs -u              # rebuild initramfs for the current kernel
sudo update-initramfs -u -k all       # rebuild for every installed kernel
sudo update-initramfs -c -k <version> # create a missing initramfs for a specific kernel
```

`update-grub` and `update-initramfs` are independent. After a kernel change you usually need both:
initramfs first (so the image exists), then `update-grub` (so the menu entry points at it). Editing
`/boot/grub/grub.cfg` by hand is wrong on Debian/Ubuntu; edit `/etc/default/grub` and rerun
`update-grub`.

## Kernel hygiene

```bash
apt list --installed 2>/dev/null | grep linux-image
sudo apt --purge autoremove           # removes old kernels kept by autoremove policy
```

- Keep at least one known-good kernel besides the newest. Never purge the running kernel
  (`uname -r`) or the only fallback, especially on a remote or headless host.
- On Ubuntu, old kernels accumulate and can fill a small `/boot`. A full `/boot` makes
  `update-initramfs` fail mid-upgrade, leaving an unbootable kernel. Free space before, not after.
- DKMS modules (NVIDIA, VirtualBox, ZFS) rebuild per kernel. After a kernel change check
  `dkms status`; a module that did not build for the new `uname -r` is the usual "driver vanished
  after update" cause.

## Reinstall GRUB (only after confirming firmware type and ESP)

```bash
# EFI systems:
sudo grub-install --target=x86_64-efi --efi-directory=/boot/efi --bootloader-id=ubuntu
# BIOS/legacy systems (disk, not partition):
sudo grub-install /dev/sda
sudo update-grub
```

## Recovery from live media (chroot)

When the system will not boot, boot a live ISO, then mount and chroot. Mount the ESP and bind the
kernel pseudo-filesystems before chrooting, or `update-grub`/`update-initramfs` inside the chroot
will misbehave.

```bash
sudo mount /dev/sdXn /mnt              # root filesystem
sudo mount /dev/sdXm /mnt/boot/efi     # ESP, on EFI systems
for d in /dev /dev/pts /proc /sys /run; do sudo mount --bind "$d" "/mnt$d"; done
sudo chroot /mnt
# inside chroot:
update-grub
update-initramfs -u -k all
exit
for d in /run /sys /proc /dev/pts /dev; do sudo umount "/mnt$d"; done
sudo umount /mnt/boot/efi /mnt
```

- For LUKS-encrypted root, `cryptsetup open` the container first; for LVM, activate with
  `vgchange -ay` before mounting.
- `--bind` (not a fresh mount) for `/dev`, `/proc`, `/sys`, `/run` so the chroot sees live kernel
  state. Missing these is the most common reason an in-chroot `grub-install` fails silently.

## Notes

- Know the kernel metapackage and firmware type before changing anything.
- Keep one known-good kernel entry around at all times.
- Hold the GRUB menu by pressing or holding `Shift` (BIOS) or `Esc` (EFI) at boot to pick an older
  kernel when the newest will not start.
- `GRUB_TIMEOUT` and `GRUB_DEFAULT` live in `/etc/default/grub`; change there and `update-grub`.
